Lab Overview

In this lab, you will work on a compiler implementation project of your own choice. The goal is to have fun and explore more advanced compiler topics. You can work in a team of more than 3 students.

We will grade you on how interesting your ideas are, how much you got working, how elegant your design is, how well you can explain it, and how creative your solution is. But remember that it is very easy to set up a grand project but make no progress at all, so try to make sure your goals are reasonable; perhaps the best you can do is to set a minimum goal that is definitely achievable and a more ambitious goal if things go well.

Sample Ideas

Here are some ideas you may want to get started with---but, you are encouraged to pursue your own ideas.

  • Implement some language features, that are now missing from MiniJava, into your Tiger compiler (e.g., exception handling, interface, nested classes, or reflections).
  • Implement a fancier garbage collector for Tiger compiler, say, a generational collector.
  • Implement a new backend for your Tiger compiler targeting other ISAs (e.g., AArch64, RISC-V 64).
  • Implement some kinds of profiling into your Tiger compiler.
  • Implement a code generator targeting Java byte code.
  • Design and implement an experimental Java Virtual Machine (JVM). The JVM you implement should support at least the bytecode instructions you will need in the Tiger compiler.
  • Implement some form of JIT in the above JVM.
  • Design and implement a backend targeting LLVM, so that your Tiger compiler can leverage the optimizations and backends of LLVM for free.
  • Design and implement an SSA-based register allocation for your Tiger compiler.

Have fun!
